## 🚀 编译选项

### 选项1: Android Studio (推荐) ⭐⭐⭐⭐⭐

**优点**:
- ✅ 最简单易用
- ✅ 自动下载依赖
- ✅ 完整IDE支持
- ✅ 可视化操作
- ✅ 最稳定可靠

**步骤**:
1. 下载并安装 [Android Studio](https://developer.android.com/studio)
2. File → Open → 选择项目目录
3. 等待 Gradle 同步完成
4. Build → Build APK 或点击运行 ▶️

**时间**: 首次约15-30分钟（含下载依赖）

---

### 选项2: 命令行编译 ⭐⭐⭐

**优点**:
- ✅ 不需要IDE
- ✅ 适合CI/CD
- ✅ 脚本自动化

**缺点**:
- ⚠️ 需要手动安装SDK
- ⚠️ 配置较复杂
- ⚠️ 约3-5GB空间

**步骤**:
```bash
# 1. 下载Android命令行工具
wget https://dl.google.com/android/repository/commandlinetools-linux-9477386_latest.zip

# 2. 安装SDK和依赖
# (详见 编译环境说明.md)

# 3. 编译项目
./gradlew assembleDebug
```

**时间**: 首次约30-60分钟（含安装SDK）

---

### 选项3: 传输到其他设备 ⭐⭐⭐⭐

**最实用的方案！**

如果您有另一台安装了Android Studio的计算机：

**步骤**:
```bash
# 1. 打包项目
./打包项目.sh

# 2. 传输压缩包到目标计算机
# (通过USB、网络、云盘等)

# 3. 在目标计算机上解压
tar -xzf weather-app-*.tar.gz
# 或
unzip weather-app-*.zip

# 4. 用Android Studio打开
# File → Open → 选择解压后的目录

# 5. 编译运行
# Build → Build APK
```

**时间**: 约5-10分钟

---

### 选项4: GitHub Actions ⭐⭐⭐⭐

**云端编译，无需本地环境！**

**步骤**:
1. 将项目上传到GitHub
2. 创建 `.github/workflows/build.yml`
3. 推送代码，自动编译
4. 下载生成的APK

**优点**:
- ✅ 无需本地SDK
- ✅ 自动化编译
- ✅ 免费使用

详见 `编译环境说明.md`
